# Source Ingest Reference

This stage writes new X-based prompt-image pairs into the source repo only.

## Repository Boundary

- execute in: `/Users/kkkk/Desktop/awesome-gptimage2-prompts`
- primary write scope:
  - `data/gpt-image-2/*.json`
  - `assets/gpt-image-2-x-discussions/`
  - `assets/gpt-image-2-x-arena-likely/`
  - generated public outputs in the same repo
- do not write `gptimg` generated files during this stage

Canonical internal SOP:

- `private/gpt-image-2-x-ingest-sop.md`

Use that document as the detailed operating guide. This reference is the compressed version for skill execution.

## Source-Of-Truth Files

- `data/gpt-image-2/x-prompt-image-pairs.json`
- `data/gpt-image-2/x-discussions-verified.json`
- `data/gpt-image-2/x-arena-likely.json`

Generated outputs:

- `README.md`

Do not treat generated outputs as editable source.

## Asset Folders

- verified X assets: `assets/gpt-image-2-x-discussions/`
- likely X assets: `assets/gpt-image-2-x-arena-likely/`
- secondary web references: `assets/gpt-image-2-web-examples/`

Gallery records must point to local asset paths, not remote `pbs.twimg.com` URLs.

## Acceptance Rules

Publish only when all of these are true:

- the final provenance URL is an X status URL
- at least one usable image can be downloaded
- the prompt can be recovered from X or a reliable secondary reference
- the prompt-image mapping is credible
- the example is actually GPT-Image-2 related

Reject or hold:

- text-only discussion
- promptless posts with no reliable recovery path
- examples clearly generated by another model
- posts where prompt-image pairing is too ambiguous

## Deduplication Rules

Deduplicate on `post_url` before download or write.

Check against all X layers, not just the flattened pair file:

- `data/gpt-image-2/x-prompt-image-pairs.json`
- `data/gpt-image-2/x-discussions-verified.json`
- `data/gpt-image-2/x-arena-likely.json`

Helper:

```bash
python3 scripts/dedupe_seed_urls.py path/to/seeds.md
```

## Prompt Recovery Rules

Priority order:

1. visible X post text
2. expanded X page text
3. trusted secondary reference repo when the X text is truncated

Allowed normalization:

- whitespace cleanup
- restoring truncated prompt text from a trusted source
- light readability cleanup

Do not invent missing prompt parts.

## ID And Slug Rules

- grouped id: `x-<status_id>`
- pair id: `x-<status_id>-<n>`
- grouped slug: descriptive result slug
- pair slug: grouped slug plus image index suffix

Keep ids stable after publication.

## Write Order

1. grouped record
2. flattened pair records
3. newest-first sort by `published_at`
4. rebuild via `python3 scripts/build_gallery.py`

## Minimum Verification

Run:

```bash
python3 scripts/build_gallery.py
python3 - <<'PY'
import json
from pathlib import Path
root = Path('.')
pairs = json.loads((root / 'data/gpt-image-2/x-prompt-image-pairs.json').read_text())
missing = [item['local_image'] for item in pairs if not (root / item['local_image']).exists()]
print(f"pairs={len(pairs)} missing_assets={len(missing)}")
if missing:
    raise SystemExit(1)
PY
```
